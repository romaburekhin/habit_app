# Habit Tracker – AI Collaboration Rules

This project is an MVP full-stack habit tracker.

IMPORTANT:
The tech stack is NOT fixed yet.
Claude is allowed to recommend the best framework choices
based on simplicity, maintainability, and scalability.

---

# 🎯 Goal

Build a minimal but clean full-stack web app that:

- Creates habits
- Tracks daily completions
- Shows weekly progress

MVP only. No authentication.

---

# 🧠 Claude Instructions

Before implementing anything:

1. Propose a tech stack.
2. Explain why it fits MVP scope.
3. Keep it simple.
4. Avoid overengineering.
5. No microservices.
6. No premature optimization.

Wait for approval before coding.

---

# 🧱 Engineering Principles

- Clean separation of concerns
- Small, readable files
- Strong typing if language supports it
- REST API
- Simple database schema
- Date-only completion tracking (YYYY-MM-DD)

---

# 🚫 Avoid

- Auth system
- Complex state managers
- Enterprise architecture
- Feature creep

---

Claude must always:
- Implement one layer at a time
